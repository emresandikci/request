# Support

## Getting Help

If you need help using this library:

- Review the project documentation in `README.md`
- Search existing GitHub Issues
- Open a new issue with a minimal reproducible example

## What to Include in Support Requests

- Node.js version
- Package manager and version
- Operating system
- Relevant request/response snippets
- Reproduction steps

## Bug Reports vs Feature Requests

- Use bug reports for incorrect behavior or regressions.
- Use feature requests for API improvements or new capabilities.

## Security Issues

For security-sensitive reports, follow `SECURITY.md` and avoid public disclosure.
